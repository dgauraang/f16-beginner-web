Now JS

To upload an Express project to a server with a URL, we're going to use Now because it's easy to use. To install it, run

npm install -g now

Then run the now command in the root of your project, follow the prompts, and the command will return a URL.

Most hosting services are quite a bit more complicated than this, but this is plenty good to work with.

Promises

We have beeen using callbacks to handle asynchronous code, but promises are another way to work with code that takes a long time to execute.

var p = new Promise((accept, reject) => {
// do something that takes a long time
if (success) {
  accept(result);
} else {
  reject(err);
}
});
p.then((result) => console.log(result));

The code with callbacks would call a function at the end of the code, just like this does, but promises can easily be chained and passed around. They also support async/await syntax on Node:

// foo is a function that returns a promise
async function bar() {
  const result = await foo();
  console.log(result);
  return result;
}

Code after the await is put into a call to "then" for the promise. The await keyword can only be used in async functions, which can only return promises (not regular values) because by the time the function finishes, the function will have awaited a promise. The return value is automatically put into a promis. The result variable is not a promise, for example, but is put into one when bar returns.

Minifiers

Most websites do not have clean, readable JavaScript and CSS when opened in the debugger. This is (hopefully) because they run minifiers before sending the code over the internet. Minifiers make files smaller while making sure the file does the same thing. The most obvious thing that a minifier can do is remove comments and whitespace. Minifiers can also change longer variable names to shorter variable names, but not all minifiers do that.

When working on an actual site, minifiers will run on the server, like node-minify. Larger projects tend to incorporate minifiers into their build script with tools like Gulp.

Databases

Databases are how websites store data. They are a pain to work with. There is a demo of a database interface at the PHPMyAdmin site.

PHPMyAdmin is the standard way of interfacing with SQL databases. There are other kinds of databases, like MongoDB, that are much more flexible and usually do not need as much administration.
