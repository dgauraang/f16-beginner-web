Front End Javascript

Front End Javascript frameworks are used to render websites dynamically and decouple the front end and backend.

Some popular frameworks include React, Angular, and Vue. Today, we'll be talking about Vue.js because it's easy to set up and simplifies some of the concepts.

The REST API

REST, or representational state transfer, is the formal name given to the structure of the HTTP protocol and a generalization of such protocols. The idea behind REST is to create a standard of stateless operations that allow browsers and servers to interact.

If a web service (basically the backend of a website) implements REST, it is called REST-compliant. All that means is that it allows apps to manipulate its resources with text and some standard stateless operations. These operations are usually the HTTP operations (GET, PUT, POST, etc.). And stateless means that each operation is self-contained and doesn't require any information not given in the request.

Frameworks

So how does this relate to what we're doing? We can build a back end with NodeJS, ExpressJS, and a database software like MongoDB or MySQL. To some extent, Express also allows to modify the front end with EJS. However, this ties the website's look heavily to the backend and increases the work the backend has to do.

We can reduce the server workload by doing all the rendering through a Javascript front end framework. Frameworks like Angular, React, and Vue have a syntax that can be placed directly within the HTML and dynamically update based on user-defined functions.

Vue

VueJS is a front end framework that is easy to read and use. Although the core library is for view rendering, it can also support single page applications (replacing the backend with just a front end framework). We'll do a tutorial with Vue so everyone can get some hands on experience. Use this link to access the JSFiddle.

Declarative Rendering

Vue works by handling everything from an instance of the "Vue" object, which you set up in javascript attached to the HTML file. You also need to include Vue in your HTML via a script tag.

<div id="app">
  {{ message }}
</div>
var app = new Vue({
  el: '#app',
  data: {
    message: 'Hello Vue!'
  }
})

This concept is at the heart of Vue. Basically, you declare that a variable is equal to something and then you insert that variable into the HTML directly. You can also insert it by binding a property to the value of that variable.

<span v-bind:title="message">Some text.</div>

You do this by using the v-bind "directive". This means that you add the v-bind keyword and a colon in front of any HTML element property you want to tie the value of to a Vue variable.

Conditions and Loops

v-if

This directive allows you to control whether elements are displayed using booleans. You set "v-if" equal to the boolean variable you want it to rely on.

v-for

This directive allows you to include loop syntax in displaying the same HTML element multiple times with small changes. See the example in the starter code.

User Input

User input in Vue still uses the concept of Javascript event listeners, but simplifies the syntax.

<button v-on:click="reverseMessage">

Use the directive "v-on" and specify an event ("click") and finally set it equal to the name of the function that handles the event.

You can also do something called two-way binding, where you take user input and use it to change a property on the page. You would do this by tying a form field to an element on the page using the "v-model" directive.

Aside: Forms

Forms can get quite complicated, but a simple text box is easy to manage.

<input id="input"></input>

// And in the JS
var form = document.getElementById("input");
form.value; // returns text inside form

The input can be read when a submit button is clicked, inside of the eventListener of the button.
